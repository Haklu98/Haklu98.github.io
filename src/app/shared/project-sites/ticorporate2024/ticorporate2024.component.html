2024 Ticorporate Project

I was part of the development team that took on the project from Elisa during the 2024 Ticorporate. Our goal was to develop a collaboration tool that both companies could use to ease their communication and provide more transparency. This would allow the companies to work more efficiently and effectively. Over the past 4,5 months we worked with this project to meet our clients needs and put out the best possible software.

My role

I was the Project Manager and also responsible for the front-end development of the project, using Angular as the framework to bring the vision of our UI designer to life. With that said, I also worked closely with our UI/UX designer to create the best possible user experience. Over the course of this project I was able to gain a lot of experience in Angular, Typescript, HTML, CSS, and SCSS, while also getting insight on how the frontend communicates with the backend and database.

The process

We used scrum process with our team to keep on track what's going on with the project and who is doing what. This greatly increased the efficiency and quality of our work and allowed us to learn what it is like to work in a scrum team. First three sprints were week long, but after the production properly started, we had 2 week long sprints.

At the start of every sprint we had a sprint planning and we went through the backlog and prioritized the tasks we had to complete. During the sprints we had daily scrums every day at 09:15, where everyone would go through what they were working on that day. At the end of the sprint we had a review on what we were able to achieve and a retrospective where we went through what we did, what went well and what could be improved.

Sprint 1

During the first sprint, we went through the roles of the team members and discussed with the Elisa's PO on what should be done first. We came to the conclusion that the most important thing to focus on was the interviewing of the clients and getting information on what is needed if this type of software was to be introduced. During this sprint we also created the GitHub repository and created a crude version of the product backlog.

So all in all, the first sprint was mainly setting up the team for the project and coming up with questions to ask the clients. We were also able to hold the first interview during the first sprint.

Sprint 2

The second sprint started with us inviting the Elisa personnel to interviews and setting up the interview questions for them. This sprint was mainly just interviewing and gathering information from the Elisa and Jamk personnel.

Sprint 3

During the third sprint we took the feedback from the interviews and decided to do two different Figma-prototypes that would be shown to the clients on both the Elisa and Jamk side. After getting feedback on the prototypes, we were able to construct the final UI/UX prototype that would become the guideline for the project. We also designed the database structure and started creating the architecture of the software.

Sprint 4

The fourth sprint was the first sprint where we started coding and actually developing the software.

I set up the angular project and pushed it to GitHub and wrote down instructions on the Git workflow. Together with the other team members, we created the Angular infrastructure so it would be as clear as possible. Then I created the main components for the project, the navigation and routing. I also added the Privacy Policy and About Us -pages. I also implemented the use of Flowbite to our project.

During the second week of this sprint, I delved into the Angular Calendar module and started coding the yearly view -component as well as the Monthly View. We also did a visit to Elisa's office to get to know the company better. I also created a little greeting for the front page that changes depending on what time it is. During the end of the sprint, I decided to personally discard the idea of using the Angular Calendar module and instead create my own.

  • Created the Angular infrastructure with the team
  • Created the components for the project

The infrastructure within Angular:

The function that creates the base for the monthly calendar:

month.component.ts

generateMonthCalendar() {
    const startOfMonthDate = startOfMonth(this.currentMonth);
    const endOfMonthDate = endOfMonth(this.currentMonth);

    // Let's get the start and end dates of the first week and last week
    const startDate = startOfWeek(startOfMonthDate, { weekStartsOn: 1 });
    const endDate = endOfWeek(endOfMonthDate, { weekStartsOn: 1 });

    // Here we generate all days between start and end
    this.days = eachDayOfInterval({ start: startDate, end: endDate });
  }

Sprint 7

I also came across a slight problem with the yearly calendar, since it was not working as well as it could be. This meant that I'd have to fix it and that one took its own time.

Sprint 8, The Last One

This was the last sprint before the end of the course, and this project. We agreed within our team to only finesse the existing components and modules. No more features were added to the project during this sprint.

For the most part I was focused on the yearly calendar, since it had the issue that it was not presenting the events correctly. I had to figure a way to span the events to cover multiple months, should they last more than one month. This was a bit of a challenge for me, but I managed to find a solution. This solution was ngClass and a bit of TypeScript. What I did, was to make a function calculate the span of the event in months and then apply the grid span to said event. Then it would span itself on the month grid and show accordingly.

What I actually did this sprint:

  • Added a function that calculates the span of the event in months
  • Stylized the yearly calendar to follow the UI/UX documentation perfectly
  • Added icons to the yearly calendar events
  • Did the last touches with GitHub and merges
  • Went through the course report with the team and filled in the retrospective

Conclusion and retrospective
